# God Vessel MVP 実装計画（GDD v0.2準拠）

## 1. 目的
このドキュメントは `GDD.md` の MVP スコープを実装可能な単位に分解し、
**戦闘の気持ちよさ / 武器ドロップ / ビルド成立 / 融合 / 属性反応 / Co-op** を最優先で進めるための実装指針を示す。

## 2. MVPスコープ（固定）
- 1職業: 魔法剣士（片手剣 + 魔法武器）
- 1マップ（出撃型セッション）
- 1ボス
- 武器10本
- 属性3種（炎・氷・雷）
- 武器融合
- 最大4人Co-op
- 拠点機能（最低限）

## 3. 非目標（MVPではやらない）
- オープンワールド
- PvP
- 大量職業・大量武器種
- 複雑な分岐ストーリー
- 建築 / クラフト中心化

## 4. 実装フェーズ

### Phase 0: プロジェクト基盤（1週間）
- Unityプロジェクト初期化
- シーン構成作成（Title / Hub / Expedition）
- 入力層・カメラ・プレイヤー移動の土台
- ScriptableObjectベースのデータ定義

**完了条件**
- ローカルで1人プレイし、Hub↔Expedition遷移ができる

### Phase 1: コア戦闘（2週間）
- 魔法剣士の通常攻撃・回避・スキル3枠
- 敵AI（近接雑魚・遠距離雑魚）
- ダメージ計算基盤（物理/属性）
- ヒットストップ・VFX・SE最小実装

**完了条件**
- 3分間の戦闘ループが破綻なく動く

### Phase 2: 属性反応（1週間）
- 属性: 炎/氷/雷
- 反応例:
  - 炎 + 風 の代替として、MVPでは `炎スタック爆発`
  - 氷 + 雷 = `凍結麻痺`
- 状態異常スタック管理

**完了条件**
- 同一敵に複数属性を当てたとき反応ログと演出が発火

### Phase 3: ドロップ/装備/ビルド（1週間）
- 武器10本（レアリティ付き）
- ランダムドロップテーブル
- 装備画面（メイン武器 + スキル3）
- ラン中レベルアップ選択（3択）

**完了条件**
- 1ラン中にビルド差が体感できる

### Phase 4: 武器融合（1週間）
- Hubで融合UI
- 継承ルール: 主要効果1 + 副効果1 + 属性1
- 融合回数制限/競合チェック

**完了条件**
- 任意の2武器から融合武器が生成され、次ランへ持ち込める

### Phase 5: Co-op（2週間）
- Steam P2P接続（最大4人）
- プレイヤー同期（位置/アニメ/攻撃イベント）
- 蘇生/全滅失敗フロー
- 主要システムのホスト権限設計

**完了条件**
- 2〜4人で1セッション完走できる

### Phase 6: ボス + 縦切り完成（1週間）
- 1ボス実装（2フェーズ）
- 報酬/帰還/Hub強化のループ接続
- チュートリアル最小導線

**完了条件**
- 「準備→出撃→戦闘→ボス→帰還→融合→再出撃」が1本通る

## 5. システム設計（最小）

### 5.1 データ定義
ScriptableObjectで以下を管理:
- WeaponData
- SkillData
- EnemyData
- ReactionRuleData
- FusionRuleData

### 5.2 実行時モデル
- RuntimeStats
- StatusEffectInstance
- LoadoutState
- RunProgress

### 5.3 戦闘計算式（暫定）
```
finalDamage = (basePower * attackScale) * (1 + elementalBonus) * critMultiplier - defense
```
- クリティカル・属性補正・防御を単純化して先にテンポを作る

## 6. 開発タスク優先度

### S（最優先）
1. 移動/攻撃/回避の手触り
2. 敵ヒットリアクション
3. 属性反応の視覚化
4. 武器ドロップと差別化
5. 融合ループ
6. Co-op同期安定化

### A（高）
1. ボス戦演出
2. Hub導線
3. 永続成長の最小要素

### B（中）
1. Lore断片
2. 追加イベント

## 7. 受け入れ基準（Definition of Done）
- 30〜60分セッションが成立
- 2名以上のCo-opで重大進行不能バグなし
- 3種類以上のビルド差を検証プレイで確認
- 融合で「強くなった実感」が得られる

## 8. リスクと対策
- **同期不整合**: 判定をホスト集約、クライアント予測は最小化
- **スコープ肥大**: 追加要素はPhase 6完了まで凍結
- **ビルド単調化**: 武器固有効果と属性反応の掛け算を優先調整
